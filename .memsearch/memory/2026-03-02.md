## Hook behavior and logic (concise)
- Hook signature / pattern:
  - "experimental.chat.system.transform": async (input, output, ctx) => { ... }
- Core flow:
  1. Extract the latest user message from `input.messages` (the most recent user role message).
  2. Load plugin config via `loadConfig(ctx.directory)`.
  3. Build a semantic search query from the user message.
  4. Perform a two-tier search:
     - Tier 1: Search the current project collection (use `--collection` and `filter: source starts_with "{projectPath}"`).
     - Tier 2 (fallback/optional): If project results < 3 (or query seems general), search the global collection (`memsearch_global`).
  5. Filter and post-process results:
     - Apply a confidence threshold (default: 0.01).
     - Deduplicate results by `chunk_hash`.
     - Rerank/trim as needed; respect token limits.
  6. Format up to 7 top relevant results (source, heading, preview).
  7. Inject into the system prompt using the `<memsearch-context>` tag via `output.system.push()` (system message).
  8. Graceful degradation: if memsearch CLI is missing, the search command errors, or there are no relevant results above threshold, the hook silently does nothing so the chat session is not blocked.

---

## Key configuration / thresholds / limits
- Confidence threshold: 0.01 (results below this are ignored).
- Deduplication key: `chunk_hash`.
- Maximum injected results: up to 7 (keeps token usage bounded).
- Project-filter pattern: `source starts_with "{projectPath}"` to prioritize project-local memories.
- Global collection name used (when needed): `memsearch_global`.
